## 2. 시장 분석 (Market Analysis)

### 2.1 Web3 게임 시장 개요

블록체인 게임 시장은 토큰 기반 경제와 디지털 자산 소유권 개념을 결합하며 빠르게 성장 중입니다. 특히 **HTML5 기반 캐주얼 게임**은 별도 설치 없이 모바일 웹뷰에서 즉시 실행 가능하여, Web3 게임 카테고리 중 **모바일 진입 장벽이 가장 낮은 영역**입니다.

기존 Web3 게임은 각자 분리된 dApp으로 존재해 사용자가 매번 지갑 연결, 토큰 교환, 온보딩을 반복해야 하는 마찰이 큽니다. TOTT는 이 문제를 **단일 허브 + 단일 토큰 경제**로 해결합니다.

### 2.2 DeFi 시장 통계

DeFi 부문은 결제 및 인프라 분야를 중심으로 기하급수적으로 성장하고 있습니다.

| 항목 | 수치 |
| --- | --- |
| 시장 규모 (2023) | $14.35 Billion |
| 시장 규모 전망 (2032) | $450 Billion 이상 |
| CAGR | 47.8% 이상 |
| 블록체인 기술 부문 (2032) | $200 Billion 이상 |
| 북미 시장 점유율 (2023) | 35% 이상 |
| 고성장 트리거 부문 | 결제 (CAGR 50% 전망) |

결제 부문의 50% CAGR은 **고빈도 트랜잭션을 발생시키는 게임 허브를 통해 DeFi 진입 관문을 선점**하겠다는 TomaTok의 전략적 근거를 제공합니다.

### 2.3 메신저 시장의 한계

WhatsApp이 DAU 12.2억·MAU 13.5억으로 글로벌 메신저 시장을 주도하고 있으나, 카카오톡·라인 등 지역 메신저는 **"우물 안 개구리(Frog in a Well)"** 현상에 갇혀 있습니다. 자국 시장의 높은 포화도에도 불구하고 언어·기술 사일로로 인해 글로벌 유동성과 국경 간 통합에 실패하는 구조입니다.

TomaTok은 AI 기반 120개 언어 실시간 번역 메신저를 **게임 허브의 글로벌 사용자 획득(UA) 및 커뮤니티 형성 채널**로 동시에 활용하여 이 한계를 돌파합니다.

## 3. 사업 모델 및 재단 (Business Model & Foundation)

### 3.1 BlockStars Foundation

홍콩에 본부를 둔 BlockStars Foundation은 DeFi 혁신과 Web3 게임 생태계 확장을 위한 R&D 주체입니다. 재단의 핵심 미션은 탈중앙화 자산 투자 및 청산을 통해 사용자에게 권한을 부여하고, 디지털 금융 연대의 미래를 구축하는 것입니다.

### 3.2 전략적 파트너십: IBOMB

TomaTok의 운영은 프로젝트의 주요 관리·R&D 파트너인 IBOMB가 담당합니다. IBOMB는 다음을 수행합니다.

- 글로벌 플랫폼 운영 및 관리
- 국내 R&D 협력 및 기술 동맹
- BlockStars 플랫폼 사업 전략 실행

### 3.3 개발 파트너 및 기술 이력

TomaTok은 대규모 엔터프라이즈 및 정부 애플리케이션에서 검증된 개발진에 의해 구축되어, 기관급 신뢰성을 보장합니다.

- **핀테크**: KT멤버십, KT Pay
- **엔터프라이즈/공공**: 기아 오토Q, 관세 교육원, 주비스 다이어트
- **특화 분야**: 육군 항공 지원 앱, 튠젬(K-Pop SNS)

## 4. 주요 서비스 (Main Services)

### 4.1 블록체인 게임 애그리게이터 허브 (메인 서비스)

TomaTok의 핵심은 **외부 개발자가 제출한 HTML5 게임을 webview로 임베드하여 단일 앱에서 통합 제공하는 게임 허브**입니다.

- 모든 게임이 **TOTT 단일 토큰**으로 결제·인게임 화폐 교환·플랫폼 정산 처리
- 사용자는 별도 설치나 지갑 연결 없이 즉시 플레이 가능
- 개발자에게는 글로벌 유저풀, 토큰 인프라, 결제 SDK를 제공하는 **B2B2C 플랫폼**
- Solana의 고성능·저지연 처리량을 활용해 게임 트랜잭션의 고빈도 처리에 최적화

### 4.2 다국어 채팅 메신저

게임 허브의 사용자 획득 및 리텐션 도구로 작동하는 글로벌 커뮤니티 채널입니다.

- 120개 언어 실시간 번역
- Voice-to-Text(V2T) 음성 노트 즉시 번역
- 그룹 채팅 내 동시 다국어 번역으로 게임 길드·커뮤니티 형성 지원

### 4.3 멀티체인 DeFi 지갑

플랫폼에서 보유한 토큰의 외부 출금·교환 게이트웨이입니다.

- Solana 기반 자산 관리
- 스테이킹·커스터디 통합
- NFT 마켓플레이스 및 토큰 간 호환성 확장 예정

### 4.4 BaaS (Blockchain as a Service)

게임 개발자가 TomaTok 플랫폼에 빠르게 통합할 수 있도록 지원하는 개발 인프라입니다.

- TOTT 결제·인게임 정산 콜백·스코어 검증을 위한 SDK
- AI 기반 보안 알고리즘 (B2B/B2C 트랜잭션 무결성 보장)
- 개발자 라이브러리를 통한 생산성 극대화 및 출시 시간 단축

## 5. 프로젝트 전략 (Project Strategy)

### 5.1 기술 스택

Solana를 기반으로 글로벌 소셜 네트워크와 DeFi 레이어를 동시에 지원하는 데 필요한 **고성능·저지연·확장성**을 확보합니다. 게임의 고빈도 트랜잭션 처리에 특히 적합합니다.

### 5.2 게이미피케이션 (Gamification)

TomaTok은 사용자의 단순 콘텐츠 소비를 **참여형 행동(action)**으로 전환하기 위해 게이미피케이션 메커니즘을 플랫폼 전반에 적용합니다.

- **포인트·레벨 시스템**: 게임 플레이, 출석, 친구 초대, 광고 시청 등 사용자의 활동이 포인트로 환산되며, 누적 포인트에 따라 사용자 레벨이 상승합니다
- **데일리·위클리 미션**: 반복적인 접속과 다양한 게임 탐색을 유도하는 일일/주간 미션 구조
- **업적·뱃지**: 특정 게임 카테고리 완수, 마일스톤 달성에 대한 TOTT 보상으로 장기 참여 유도

게이미피케이션은 **게임 자체뿐 아니라 플랫폼 사용 행위 전반을 게임화**함으로써, 단일 게임에 의존하지 않는 사용자 리텐션 구조를 형성합니다.

### 5.3 GameFi 통합 모델

GameFi는 게임(Game)과 탈중앙 금융(DeFi)이 결합된 개념으로, 게임 내 경제 활동이 온체인 자산과 직접 연결되는 구조를 의미합니다. TomaTok은 다음 세 축으로 GameFi 모델을 구현합니다.

| 축 | 내용 |
| --- | --- |
| **자산화** | 게임 내 활동 결과(점수, 보상, 아이템)가 TOTT 또는 NFT로 변환 가능 |
| **유동성** | 사용자는 보유 자산을 외부 거래소·DEX·DeFi 프로토콜에서 자유롭게 활용 가능 |
| **참여형 거버넌스** | TOTT 홀더가 게임 등록 정책, 노출 가중치, 정산 비율 등 핵심 의사결정에 참여 |

기존 GameFi 프로젝트들이 단일 게임에 종속된 토큰 경제를 가졌던 것과 달리, TomaTok은 **다수의 게임이 단일 토큰(TOTT)과 단일 사용자 풀을 공유하는 허브 모델**을 통해 개별 게임 사이클의 변동성을 평탄화합니다.

### 5.4 TOTT 활용 시나리오

TOTT는 TomaTok 생태계 내 모든 가치 이동의 기축 통화로 기능합니다. 주요 활용 시나리오는 다음과 같습니다.

**사용자 측면**

- 인게임 아이템·재화 결제
- 게임 내 활동 보상 수령
- 프리미엄 콘텐츠·기능 잠금 해제
- 메신저 내 P2P 송금 및 선물하기
- 외부 거래소를 통한 자산 환금

**개발자 측면**

- 게임 등록 수수료 납부
- 인게임 매출 정산 수령 (주간 또는 월간)
- 프로모션 슬롯 입찰
- 광고 노출 정산

**플랫폼 측면**

- 거버넌스 투표권 행사
- 트랜잭션 수수료 결제
- 스테이킹을 통한 온체인 참여

이러한 다층적 활용 구조는 TOTT가 **단일 용도가 아닌 생태계 전반의 결제·보상·거버넌스 매개체**로 기능함을 의미합니다.

### 5.5 P2E (Play-to-Earn)

P2E는 사용자가 게임 플레이를 통해 토큰 또는 토큰화 가능한 자산을 획득하는 모델입니다. TomaTok 게임 허브에 등록된 게임 중 P2E를 채택한 게임은 다음 원칙을 따릅니다.

- **활동 기반 보상**: 사용자의 실제 플레이 활동(점수, 미션 완수, 랭킹 달성)에 따라 보상이 지급됩니다
- **발행 한도 통제**: 각 게임의 일일·주간 보상 발행량은 BlockStars Foundation 심사 위원회의 검토를 거쳐 설정되며, 인플레이션을 통제합니다
- **어뷰징 방지**: 봇·다중 계정·비정상 패턴은 온체인·오프체인 모니터링을 통해 차단됩니다
- **선택적 채택**: 모든 게임이 P2E를 채택할 필요는 없으며, 일반 게임도 동일한 허브 내에서 공존합니다

P2E는 게임 플레이 시간을 가치 창출 활동으로 전환하지만, **보상의 지속 가능성은 게임의 인기도, 토큰 시장 상황, 발행 정책 등 복합 요인에 의해 결정**되며, 어떠한 수익도 보장되지 않습니다.

### 5.6 T2E (Tap-to-Earn)

T2E는 TomaTok이 자체 운영하는 **탭(터치) 기반 캐주얼 게임을 통해 사용자가 포인트를 획득하고, 누적 포인트를 TOTT로 전환**할 수 있는 모델입니다.

| 단계 | 내용 |
| --- | --- |
| **1. 참여** | 사용자가 TomaTok 내장 T2E 게임(탭 게임, 미니 챌린지)에 접속 |
| **2. 포인트 획득** | 게임 플레이를 통해 플랫폼 포인트 적립 |
| **3. TOTT 전환** | 누적 포인트가 일정 기준 도달 시 TOTT로 전환 가능 |
| **4. 활용** | 전환된 TOTT는 인게임 결제, 송금, 외부 거래소 출금 등에 사용 가능 |

T2E의 특징은 다음과 같습니다.

- **낮은 진입 장벽**: 별도의 학습이나 투자 없이 누구나 즉시 참여 가능
- **광고 연동**: 일부 T2E 게임은 광고 시청을 통해 추가 포인트를 제공하며, 광고 매출이 보상 풀의 일부를 구성합니다
- **부정 사용 방지**: 시간당 획득 한도, 디바이스 검증, 행동 패턴 분석을 통해 자동화된 어뷰징 차단
- **온보딩 도구**: T2E는 Web3에 익숙하지 않은 사용자가 TOTT 토큰과 지갑 사용에 자연스럽게 입문하는 **게이트웨이 게임**으로 기능

T2E의 포인트-TOTT 전환 비율, 일일 획득 한도, 출금 조건은 플랫폼 운영 데이터에 따라 조정될 수 있으며, 변경 시 사전 고지됩니다.

### 5.7 사용자 참여 중심 경제

TomaTok은 단순 인센티브 모델보다 지속 가능한 **사용자 참여 중심 경제**를 지향합니다.

- **자기 관리(自己 管理)**: 사용자가 자신의 소비·저축 습관을 직접 관리할 수 있도록 권한 부여
- **자기 중심 소비**: 플레이, 학습, 커뮤니티 활동 등 사용자의 플랫폼 참여 데이터를 기반으로 개인화된 서비스와 유틸리티를 제공

위 5.5 P2E와 5.6 T2E는 본 사용자 참여 경제의 구체적 실행 모델이며, 사용자가 자신의 활동 데이터를 직접 통제하고 그 가치를 회수할 수 있는 구조를 제공합니다. **게임 허브가 사용자 참여 경제의 핵심 실행 레이어**로 기능합니다.

### 5.8 플랫폼 트리거

TomaTok은 다음 경로를 통해 DeFi 채택의 "도화선" 역할을 수행합니다.

- **글로벌 시장 점유**: 게임 허브의 고빈도 트랜잭션을 통한 사용자 확보
- **토큰 거버넌스**: TOTT를 생태계의 기축 유틸리티로 확립
- **자산 관리 다양성**: 다양한 탈중앙화 금융 수단 제공

## 7. 스마트 컨트랙트 및 온체인 스테이킹 프로토콜 (Smart Contract & Staking Protocol)

TOTT 스테이킹 프로토콜은 Solana 기반 Anchor 프로그램으로 구현되어 있습니다. 사용자의 스테이킹 포지션, 처리 상태, 계정 정보는 온체인 PDA(Program Derived Address) 계정에 기록됩니다.

본 섹션은 온체인 식별자와 기술 구조를 설명하기 위한 목적이며, 토큰 가치, 경제적 이익, 지급 지속성, 또는 특정 결과를 보장하지 않습니다.

### 7.1 온체인 식별자

| 항목 | 주소 / 값 |
| --- | --- |
| **Staking Program ID** | `8UzWbsBj6EgZVmFaFFqHowqD4QzRYH5UaZWPvAuVV6X` |
| **Pool Initialization Authority** | `85Etjjx3kj8fyJUdqe3Bz4rN2fDjA7L9sWndtumpAPUa` |
| **StakingPool PDA** | `BXgqKF9Vgfcej3jMLvBgy66E4JUXcLehA8QJ4KFimGKD` |
| **TokenVault PDA** | `CKPToXHH1PEbH9Wuwra9cBwheqST7UKBZL5WteZy5Uph` |
| **Token Program** | `TokenkegQfeZyiNwAJbNbGKPFXCWuBvf9Ss623VQ5DA` |

사용자는 Solana Explorer에서 Token Mint, Staking Program ID, StakingPool PDA, TokenVault PDA를 직접 조회하여 온체인 상태를 확인할 수 있습니다.

### 7.2 계정 구조와 PDA 모델

Solana 프로그램은 실행 로직과 상태 저장 계정을 분리합니다. TOTT 스테이킹 프로토콜은 다음 PDA 계정 구조를 사용합니다.

| 계정 | PDA Seed | 역할 |
| --- | --- | --- |
| **StakingPool** | `["staking_pool", token_mint]` | 풀 전역 상태, 총 스테이킹 원금, 토큰 볼트 주소 기록 |
| **TokenVault** | `["token_vault", token_mint]` | 스테이킹된 TOTT가 보관되는 프로그램 통제 토큰 계정 |
| **UserStake** | `["user_stake_v4", user, staking_pool, index]` | 사용자별 개별 스테이킹 포지션 기록 |

`index` 값을 사용함으로써 동일 사용자는 여러 개의 독립적인 스테이킹 포지션을 동시에 운용할 수 있습니다.

### 7.3 스테이킹 포지션 및 시간 기준

프로토콜은 스테이킹 포지션의 기간 계산을 위해 30/360 시간 기준을 사용합니다. 즉, 1개월은 30일, 1년은 360일로 계산됩니다.
지원되는 스테이킹 기간은 6개월, 9개월, 12개월입니다. 각 포지션은 별도 `UserStake` 계정으로 관리되며, 시작 시각, 잠금 기간, 마지막 처리 시각 등의 상태값을 온체인에 저장합니다.

### 7.4 주요 온체인 액션

| 액션 | 설명 |
| --- | --- |
| **풀 초기화** | 관리자 지갑으로 StakingPool 및 TokenVault PDA 생성 |
| **스테이킹** | 사용자 TOTT를 TokenVault로 전송하고 UserStake 계정 생성 |
| **포지션 처리** | 일정 기간 경과 후 포지션 상태를 온체인 규칙에 따라 처리 |
| **언스테이킹** | 만기 또는 중도 언스테이킹 처리 |
| **계정 정리** | 언스테이킹 완료 후 PDA 계정을 닫고 rent 반환 |

실패한 온체인 트랜잭션은 플랫폼 데이터베이스에 정상 스테이킹 기록으로 반영되지 않습니다.

### 7.5 안전장치 및 리스크 관리

| 안전장치 | 설명 |
| --- | --- |
| **Vault 잔고 검증** | TokenVault 잔고가 필요한 처리 금액을 충족하는지 검증 |
| **오버플로우 방어** | 주요 수학 연산에 checked 연산과 u128 중간 계산 사용 |
| **소유권 검증** | `has_one = user` 제약을 통해 타인의 UserStake 조작 방지 |
| **시간 검증** | 현재 시간이 스테이킹 시작 시각 또는 마지막 청구 시각보다 앞서는 비정상 상태 차단 |
| **토큰 계정 검증** | 관련 토큰 계정의 mint, owner, frozen 상태 검증 |

위 안전장치는 프로토콜 규칙의 무결성을 높이기 위한 기술적 장치입니다. 다만 스마트 컨트랙트, 지갑, 네트워크, 시장 가격, 유동성, 규제 변화와 관련된 일반적인 Web3 리스크는 여전히 존재합니다.

## 8. 시스템 아키텍처 및 성장 엔진 (System Architecture & Growth Engine)

TomaTok은 글로벌 고가용성을 위해 설계되었으며, 기술 인프라와 공격적인 사업 확장을 결합합니다.

## 10. 게임 노출 (Exposure & Ranking)

등록된 게임은 TomaTok 앱 내 다양한 노출 슬롯에 표시됩니다.

### 10.1 노출 슬롯 구조

| 슬롯 | 위치 | 설명 |
| --- | --- | --- |
| **메인 배너** | 홈 최상단 | 추천·신작·이벤트 게임 (5~10개 롤링) |
| **카테고리별 랭킹** | 홈 상단 탭 | 장르·언어·지역별 Top 10 |
| **신작 (New)** | 홈 중단 | 등록 30일 이내 게임 |
| **인기 급상승 (Trending)** | 홈 중단 | 24~72시간 활성도 급증 게임 |
| **프로모션 슬롯** | 광고 영역 | 개발자 TOTT 입찰 기반 노출 |

### 10.2 노출 정책

- **공정성**: 동일 카테고리 내 무광고 슬롯과 광고 슬롯의 시각적 구분 명시
- **품질 기준**: 평점 2.5 이하·이상 거동(어뷰징) 감지 시 노출 제한
- **다양성**: 단일 개발자가 메인 배너의 30%를 초과 점유 불가
- **다국어 우선**: 사용자 언어 설정에 맞춘 게임 우선 노출
- **거버넌스**: 노출 정책 변경은 TOTT 홀더 거버넌스 절차를 통해 결정
